Лабораторная работа: Графовые базы данных в MS SQL Server
Вариант 51. Тема: Футбол — Игроки, клубы, трансферы (история переходов).

Условие задачи
==============

Спроектировать и реализовать графовую базу данных FootballGraph для
моделирования профессионального футбола. База данных должна хранить
информацию об игроках, клубах, странах, истории трансферов (переходов
игроков между клубами), гражданстве игроков и принадлежности клубов
к странам.

Состав графа
------------

Таблицы узлов (3 шт.):
  1) Player   — футболист (id, имя, амплуа, год рождения, гражданство).
  2) Club     — футбольный клуб (id, название, страна, год основания).
  3) Country  — страна (id, название, континентальная конфедерация).

Таблицы рёбер (3 шт., все направленные, с атрибутами):
  1) TransferredTo (Player -> Club) — трансфер игрока в клуб.
     Атрибуты: дата трансфера, сумма (млн €), длина контракта,
     тип трансфера (purchase / loan / free).
     Один игрок может многократно переходить в один и тот же клуб —
     каждый переход хранится отдельным ребром.
  2) ClubFromCountry (Club -> Country) — клуб базируется в стране.
     Атрибут: год основания клуба в этой стране.
  3) PlayerNationality (Player -> Country) — гражданство игрока /
     принадлежность к национальной сборной.
     Атрибуты: количество матчей за сборную, голов за сборную.

Все таблицы рёбер должны иметь CONNECTION CONSTRAINT, ограничивающий
типы соединяемых узлов.

Заполнение
----------

Каждая таблица узлов содержит не менее 10 строк.
Таблицы рёбер заполняются так, чтобы реализовалась реальная
(приближённая) хронология переходов известных игроков, например:
  Messi:    Barcelona -> PSG -> Inter Miami
  Ronaldo:  Man United -> Real Madrid -> Juventus -> Man United -> Al-Nassr
  Neymar:   Santos -> Barcelona -> PSG
  Mbappe:   PSG -> Real Madrid
  Ibrahimovic: Juventus -> Barcelona -> Milan -> PSG -> Man United
  и т.д.

Запросы
-------

Реализовать с использованием функции MATCH не менее 5 запросов,
содержащих цепочки из 3 и более узлов, в том числе:
  - история всех клубов конкретного игрока;
  - игроки, игравшие в одном и том же клубе вместе с заданным игроком;
  - игроки-легионеры (играющие в клубах не своей страны);
  - самые дорогие трансферы в клубы выбранной страны;
  - игроки, играющие в клубе своей страны.

Реализовать с использованием функции SHORTEST_PATH не менее 2 запросов
(с использованием шаблонов "+" и "{1,n}") с выводом пройденного пути
через STRING_AGG и LAST_VALUE, например:
  - все клубы, достижимые из заданного игрока через сеть трансферов;
  - цепочки трансферов длиной от 1 до 4 от заданного игрока.

Дополнительно
-------------

  - Диаграммы графа выполнены в diagrams.net (.drawio) на отдельных
    страницах: схема узлов, схема всех рёбер.
  - Граф визуализирован в Power BI с помощью Force-Directed Graph
    (история трансферов и/или гражданства).
